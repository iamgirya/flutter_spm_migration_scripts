# `spm_plugin_migration.dart`

Скрипт **автоматической маршрутизации** одного Flutter-плагина с классической iOS-вёрстки CocoaPods (`ios/Classes`, assets, podspec) к **совместимой с Swift Package Manager** структуре по [гайду Flutter для авторов плагинов](https://docs.flutter.dev/packages-and-plugins/swift-package-manager/for-plugin-authors), при этом **обновляет podspec**, чтобы CocoaPods продолжал находить источники в новых путях.

## Запуск

```text
dart run spm_plugin_migration.dart /абсолютный/путь/к/корню/плагина
```

Требуется ровно один аргумент — существующий каталог плагина с подкаталогом `ios/` и как минимум одним `.podspec` в `ios/`.

## Что делает по шагам (логика)

1. **Находит podspec**, из имени файла берёт `pluginName` и производное имя библиотеки для SPM (`_` → `-`).
2. **Читает podspec**: минимальная iOS-версия, зависимости CocoaPods (кроме `Flutter` выводятся как NOTICE — их руками переносят в SPM), имена resource bundles.
3. **Определяет язык**: по файлам в `ios/Classes` и уже существующему `ios/<plugin>/Sources/<plugin>` — чистый Swift, Objective-C или смешанный; для смешанного выдаёт NOTICE про возможное разделение таргетов SPM.
4. **Обновляет пути Pigeon** (если используются) под новый layout.
5. **Создаёт дерево** `ios/<pluginName>/Sources/<pluginName>/`, при необходимости `.swiftpm`/`arc`-игноры для Swift.
6. **Переносит каталоги**: `Assets/` и `Resources/` → внутрь `Sources/<plugin>/`; `Classes/` → туда же; при нахождении `PrivacyInfo.xcprivacy` — в ожидаемое место под SPM.
7. **Objective-C**: раскладывает публичные заголовки и `modulemap` в `include/<plugin>/` согласно правилам podspec и эвристикам; при смешанном плагине может **автоматически разделить** на таргеты `<plugin>` (Swift) и `<plugin>_objc` с предупреждением про `import …_objc`.
8. **Swift**: после переноса при необходимости добавляет `import Flutter` в файлы, использующие типы Flutter.
9. **Генерирует `Package.swift`** (если файла ещё нет — иначе WARN и пропуск записи): таргеты, ресурсы, приватный манифест, зависимости из podspec как заготовка/комментарии по ситуации.
10. **Переписывает импорты в ObjC** на новые пути `include/`, помечает вызовы имён CocoaPods resource bundles комментариями TODO для ручной правки под `Bundle.module`.
11. **Обновляет podspec** (пути `source_files`, headers, modulemap и т.д.) для совместимости с новым деревом.
12. **Дополнительно**: пример приложения, интеграция с `arc` (VCS), очистка пустых старых каталогов.

Скрипт рассчитан на **идемпотентные повторные запуски** в части переносов и правок layout (с учётом уже сдвинутых файлов).

## Ограничения

- Зависимости, объявленные только в CocoaPods, автоматически в SPM не портируются — только списки и предупреждения.
- Существующий `Package.swift` не перезаписывается (только предупреждение).
- В шапке файла — пометка о происхождении кода; при доработках ориентируйтесь на поведение в репозитории, а не только на комментарий.
